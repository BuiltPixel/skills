# Security Policy

## Supported Versions
We release patches for security vulnerabilities in actively maintained versions.  
Only the latest stable release is officially supported.

| Version | Supported |
| ------- | --------- |
| Latest  | ✅        |
| Older   | ❌        |

## Reporting a Vulnerability
If you discover a security vulnerability in this repository, please help us by reporting it responsibly.

- **Do not open a public issue** describing the vulnerability.
- Instead, send a detailed report to our security contact: **InserloftSupport@gmail.com**.
- Include:
  - Steps to reproduce the vulnerability
  - Potential impact
  - Suggested mitigation (if available)

We will:
1. Acknowledge receipt of your report within 72 hours.
2. Investigate and validate the issue.
3. Provide updates on remediation progress.
4. Credit you in the release notes if you wish.

## Security Best Practices
To keep your environment safe when using this project:
- Always use the latest release.
- Avoid running untrusted or modified code.
- Review dependencies for known vulnerabilities.
- Follow the principle of least privilege when deploying.

## Responsible Disclosure
We encourage responsible disclosure. Please give us reasonable time to fix issues before making them public.  
We are committed to transparency and will publish advisories for confirmed vulnerabilities.
